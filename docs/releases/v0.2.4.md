# Console v0.2.5 — Identity & Library Foundations

## Added

- Renamed Resources to Library throughout the main Console workflow.
- Added the new `library` Firestore collection model.
- Added File and Link item types.
- Added source, category, version and status metadata.
- Added Internal, All Customers and Selected Customers visibility.
- Added multi-customer selection.
- Changed Storage paths so files are independent of projects and customers.
- Added permanent project, roadmap, architecture, Firebase, UI, coding and testing documentation.
- Added architectural decision records.

## Changed

- Creating a Library item no longer requires a customer or project.
- Project counters are no longer updated by new Library items.

## Important migration note

Existing v0.2.3 `resources` records are left untouched. They need a deliberate migration decision because their correct new visibility cannot be safely assumed.
