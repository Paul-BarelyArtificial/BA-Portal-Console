# v0.2.6a – Customer Portal Invites

## Purpose
Give administrators a way to create a customer's Portal login directly from the Console. Previously, adding a customer in the Console created no way for them to actually sign in to the Portal.

## Changes
- Added a "Send Portal invite" action to the customer detail panel.
- The Console creates the customer's Firebase Authentication account using their contact email, then Firebase emails them a link to set their own password (using Firebase's built-in password-set page — no new Portal page required).
- Account creation runs through a temporary, isolated secondary Firebase app instance so the administrator's own signed-in session is never affected.
- The administrator never sees or sets the customer's password.
- Pressing the button again after an account exists resends the password-set email rather than erroring.
- Added `portalAccountCreated` and `portalInviteSentAt` fields to `customers` records; invite status is shown in the customer detail panel.
- Added the customer's contact name and contact email to the customer detail panel (View button) — previously neither was shown there at all.

## Fix included in this release
- Cache-busting query strings on the Console's CSS and JS files were still `?v=0.2.5` / `?v=0.2.3` despite the underlying code being updated for v0.2.6. This could leave a browser that had previously visited the Console stuck running old code. All Console asset references are now `?v=0.2.6a`.

## Firebase changes
None to Firestore rules or collections beyond the two new fields on `customers` (already covered by existing admin write access). No change required to Firebase Authentication settings — Email/Password sign-in was already enabled for Console admin login.

## Testing
1. Sign in as an administrator.
2. Open Customers and create (or open) a customer with a contact email you can receive mail at.
3. Open the customer detail panel and confirm it shows "Portal login: Not set up".
4. Click "Send Portal invite". Confirm the button briefly disables and a status message appears.
5. Confirm the panel now shows "Portal login: Invite sent" with a date, and the button now reads "Resend Portal invite".
6. Check that inbox for a Firebase password-set email; follow the link and set a password.
7. Sign in to the Portal with that email and the new password and confirm access.
8. Click "Resend Portal invite" again and confirm a second email arrives without error.
9. Confirm your own administrator session in the Console was not signed out at any point during this test.
10. Hard-refresh the Console (Cmd+Shift+R) and confirm the version footer reads `v0.2.6a – Customer Portal Invites`.
