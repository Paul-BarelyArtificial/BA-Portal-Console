# v0.2.6b – Customer Edit and Archive

## Purpose
Previously, Customers, Projects and Library could each be created but never edited or removed. This is the first of three releases (Customers, then Projects, then Library) adding Edit and Archive. Bookings is deliberately excluded from this series — it is still sample data, not a live Firestore-backed feature yet, so it needs its own separate piece of work before Edit/Delete makes sense there.

## Changes
- "Edit customer" on the customer detail panel reopens the New Customer dialog pre-filled with that customer's details. Saving updates the existing Firestore record rather than creating a new one.
- "Archive customer" / "Reactivate customer" sets the customer's status to Archived (or back to Active), after a confirmation prompt. This is a soft delete: the record is never removed, so Projects and Library items that reference this customer are unaffected.
- Added "Archived" as a customer status, in both the status dropdown and the customer filter buttons.

## Why archive instead of delete
A permanent delete would leave Projects and Library "Selected Customers" assignments pointing at a customer that no longer exists. Archiving keeps the record (and its history) intact while making clear it's no longer active — the same approach already used for Projects and Library status.

## Firebase changes
None. Uses the existing `customers` collection and existing admin write access.

## Testing
1. Sign in as an administrator and open Customers.
2. Open an existing customer and click "Edit customer". Confirm the dialog opens pre-filled with the current details and the button reads "Save Changes".
3. Change a field (e.g. notes) and save. Confirm the change appears in the table and detail panel without a duplicate record being created.
4. Click "New Customer" and confirm the dialog opens empty with "Create Customer" as the button (i.e. edit state doesn't leak into new-customer creation).
5. Open a customer and click "Archive customer". Confirm the confirmation prompt, then confirm the status changes to Archived and the button now reads "Reactivate customer".
6. Confirm an archived customer's existing Projects and Library assignments still work exactly as before.
7. Use the new "Archived" filter button to confirm archived customers can still be found.
8. Click "Reactivate customer" and confirm the status returns to Active.

## Known issues
None known. Projects and Library will get the same Edit/Archive treatment in the next two releases.
