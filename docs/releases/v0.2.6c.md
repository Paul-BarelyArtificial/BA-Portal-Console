# v0.2.6c – Archived Customers Lose Library Access

## Purpose
v0.2.6b added Archive/Reactivate for customers but didn't define what it actually does to their Portal access. This release closes that gap.

## Behaviour
- **Login is not blocked.** Firebase Authentication accounts cannot be disabled from client-side code — that requires the Admin SDK, which only runs on a backend server (e.g. a Cloud Function), and this project doesn't have one. Adding that would be a separate infrastructure decision, not part of this release.
- **Library access is fully cut off instead.** Archiving a customer removes their `customerAccess/{email}` mapping. The Portal's Library page depends on that mapping to know who a signed-in user is; without it, they're treated as unlinked and their Library shows no items at all — this covers both "All Customers" items and any "Selected Customers" items assigned specifically to them.
- **Reactivating restores it automatically.** As soon as a customer's status changes back to anything other than Archived, the next Console sync (which already runs whenever the customer list loads) recreates their `customerAccess` mapping and their Library returns to normal — no extra step required.
- The customer detail panel now shows a note when a customer is archived, so it's clear at a glance what "Archived" actually means.

## Firebase changes
None. Uses the existing `customerAccess` collection and existing admin write (including delete) access already granted by the Firestore rules.

## Testing
1. Sign in as an administrator. Pick a test customer with a working Portal login and at least one Published Library item assigned via "All Customers" and one via "Selected Customers" naming them.
2. Sign in to the Portal as that customer in a separate session/browser and confirm both items are visible.
3. In the Console, archive that customer.
4. In the Portal session, reload Your Library. Confirm it now shows no items, and sign-in itself still works (they are not logged out or blocked).
5. In the Console, reactivate the customer.
6. In the Portal session, reload Your Library again and confirm both items reappear.
7. Confirm the customer detail panel shows the "This customer is archived…" note only while archived.

## Known issues
None known. A true "block login entirely" option would need a backend Cloud Function using the Firebase Admin SDK — a separate decision, not covered here.
