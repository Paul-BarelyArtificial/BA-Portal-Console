# v0.2.6d – Project Edit and Archive

## Purpose
Second of three releases adding Edit and Archive to the Console's core record types (Customers done in v0.2.6b/c, Library next). Projects already had an Archived status and filter button from earlier releases — this adds the actions to actually use it, plus editing.

## Changes
- "Edit project" on the project detail panel reopens the New Project dialog pre-filled with that project's details (name, status, type, description). Saving updates the existing Firestore record.
- The **customer** a project belongs to is shown but cannot be changed via Edit — it's locked in the dialog while editing. Reassigning a project to a different customer would require adjusting the project count on both the old and new customer records, which is a bigger, riskier change than this release covers. To move a project to a different customer, archive the old one and create a new project against the correct customer.
- "Archive project" / "Reactivate project" sets status to Archived (or back to Planning), after a confirmation prompt.
- Confirmed: archiving a project has **no effect on Library**. Library items are independent of projects by design (decision 001), so nothing needs to change there.

## Firebase changes
None. Uses the existing `projects` collection and existing admin write access.

## Testing
1. Sign in as an administrator and open Projects.
2. Open an existing project and click "Edit project". Confirm the dialog opens pre-filled, the customer field is visible but disabled, and the button reads "Save Changes".
3. Change the status or description and save. Confirm the change appears without creating a duplicate project or changing the customer's project count.
4. Click "New Project" and confirm the dialog opens empty with the customer field enabled again and "Create Project" as the button.
5. Open a project and click "Archive project". Confirm the prompt, then confirm status changes to Archived and the button now reads "Reactivate project".
6. Confirm the Archived filter button still finds it.
7. Click "Reactivate project" and confirm it returns to Planning.

## Known issues
None known. Library gets the same Edit/Archive treatment next.
