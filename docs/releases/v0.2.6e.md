# v0.2.6e – Library Edit, Archive and Delete

## Purpose
Last of three releases adding Edit and Archive to the Console's core record types (Customers, Projects, now Library). Also adds a genuine permanent delete for Library items specifically, to manage Firebase Storage usage — requested directly, since Library is the only one of the three where a hard delete is actually safe.

## Changes
- **Edit item**: reopens the New Library Item dialog pre-filled with title, description, source, category, version, status, visibility and selected customers. Saving updates the existing record.
- **File/link is locked during edit**: the dialog shows the current file name or link as plain text instead of an editable field. To change the actual file or link, delete the item and create a new one. This keeps the edit path simple and avoids the risk of a half-replaced upload.
- **Archive item / Reactivate item**: quick-action status toggle, reusing the Archived status Library already had.
- **Delete permanently**: a real, irreversible delete — removes the Firestore record and, for File items, the actual file in Firebase Storage. This is safe to do as a hard delete (unlike Customers/Projects) because Library items aren't referenced anywhere else — decision 001 established Library items are independent of projects and customers by design. Requires a confirmation prompt naming the item.

## Firebase changes
None. Uses the existing `library` collection, existing admin write/delete access, and existing Storage paths.

## Testing
1. Sign in as an administrator and open Library.
2. Open an existing item and click "Edit item". Confirm the dialog opens pre-filled, the file/link fields are replaced with a read-only note showing the current file name or link, and the button reads "Save Changes".
3. Change a field (e.g. description or visibility) and save. Confirm the change appears and the file/link is untouched.
4. Click "New Library Item" and confirm the dialog opens empty and normal again (file/link fields editable, correct button text).
5. Open an item and click "Archive item" / "Reactivate item" and confirm the status toggles correctly.
6. Open a File item and click "Delete permanently". Confirm the prompt, then confirm the item disappears from the table.
7. In Firebase Console → Storage, confirm the corresponding file under `library/{itemId}/…` is also gone.
8. Repeat delete on a Link item and confirm it's removed cleanly (no Storage action expected, since there's no file).

## Known issues
None known. This completes the planned Edit/Archive/Delete work for Customers, Projects and Library. Bookings remains sample data and needs its own separate "go live" piece of work before any of this applies there.
