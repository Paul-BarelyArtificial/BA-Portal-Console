# v0.2.8 – Library Bulk Upload

## Purpose
Adding Library items one at a time got repetitive when uploading several files that share the same category, visibility and status. This adds a dedicated Bulk Upload flow, matching the roadmap's "Add multiple files efficiently, apply shared metadata and visibility" goal.

## Changes
- New "Bulk Upload" button next to "New Library Item" on the Library page, opening a dedicated dialog.
- Select any number of files at once (same allowed types and 50 MB per-file limit as single uploads).
- One set of shared settings — Category, Source, Visibility (with Selected Customers picker), Status, Version — applies to every file in the batch.
- Titles are generated automatically from each filename (extension stripped, underscores/dashes become spaces) — no per-file title entry, to keep bulk uploads fast.
- Files upload one at a time with a live progress message ("Uploading 2 of 5: filename.pdf"). If a file fails (wrong type, too large, or an upload error), it's skipped and recorded, and the rest of the batch continues — you get a summary naming exactly what failed and why, instead of the whole batch stopping.
- Bulk Upload is File-only (no Links) — that matches the roadmap scope. Use "New Library Item" for a single Link.

## Firebase changes
None. Uses the existing `library` collection, existing Storage paths (`library/{itemId}/{filename}`), and existing admin write access — no rules changes needed.

## Testing
1. Sign in as an administrator and open Library.
2. Click "Bulk Upload". Select 3–4 files of mixed types (e.g. a PDF, a Word doc, an image).
3. Set Category, Source, Visibility (try "Selected Customers" and pick a customer), Status and Version.
4. Click "Upload Files" and confirm the progress message updates per file.
5. Confirm all files appear in the Library table with the shared settings applied and titles matching their filenames.
6. Try a batch that includes one invalid file (e.g. an unsupported type, or one over 50 MB). Confirm the valid files still upload and the summary message correctly names the failed one.
7. Confirm the dialog resets cleanly (no leftover selected files or messages) the next time you open it.
